演算哲學是透過程式碼表達的運算美學運動。輸出 .md 檔案（哲學宣言）、.html 檔案（互動式檢視器）和 .js 檔案（生成演算法）。

此過程分為兩個步驟：
1. 演算哲學建立（.md 檔案）
2. 透過建立 p5.js 生成藝術來表達（.html + .js 檔案）

首先進行以下任務：

## 演算哲學建立

開始時，建立一套**演算哲學**（不是靜態圖像或範本），透過以下方式詮釋：
- 運算過程、湧現行為、數學之美
- 種子隨機性、噪聲場、有機系統
- 粒子、流、場、力
- 參數化變異與受控混沌

### 關鍵理解
- 接收的是：使用者給予的一些微妙輸入或指示，作為基礎參考；但這不應限制創作自由。
- 建立的是：一套演算哲學/生成美學運動。
- 接下來的是：同一個版本接收這套哲學，並以程式碼**表達它**——建立 90% 演算生成、10% 必要參數的 p5.js 草稿。

思考方式如下：
- 為一個生成藝術運動撰寫宣言
- 下一階段負責撰寫使其生動的演算法

哲學必須強調：演算表達、湧現行為、運算之美、種子變異。

### 如何生成演算哲學

**為運動命名**（1-2 個詞）：「有機湍流」/「量子諧波」/「湧現靜止」

**闡述哲學**（4-6 段——精簡但完整）：

為了捕捉**演算本質**，表達此哲學如何透過以下方式呈現：
- 運算過程和數學關係？
- 噪聲函式和隨機模式？
- 粒子行為和場動態？
- 時間演化和系統狀態？
- 參數化變異和湧現複雜性？

**重要指南：**
- **避免重複**：每個演算面向只提一次。避免重複關於噪聲理論、粒子動力學或數學原理的概念，除非能增加新的深度。
- **反覆強調工藝精神**：哲學**必須**多次強調最終演算法應看起來如同歷經無數小時開發、精心打磨，出自絕對頂尖的大師之手。這個框架至關重要——反覆使用如「精心打造的演算法」、「深厚運算專業的結晶」、「煞費苦心的最佳化」、「大師級實作」等短語。
- **留有創作空間**：對演算方向要具體，但足夠精簡，讓下一個 Claude 有空間以極高的工藝水準做出詮釋性的實作選擇。

哲學必須引導下一個版本以**演算方式**表達想法，而非透過靜態圖像。美存在於過程中，不在最終畫面。

### 哲學範例

**「有機湍流」**
哲學：被自然法則約束的混沌，秩序從無序中湧現。
演算表達：由多層 Perlin 噪聲驅動的流場。數千個粒子沿向量力運動，其軌跡累積成有機密度圖。多個噪聲倍頻創造湍流區域和平靜地帶。顏色從速度和密度中湧現——快速粒子燃燒成明亮，緩慢的消退成陰影。演算法運行至平衡——一個經過大師級運算美學家無數次迭代精心調整的平衡，每個參數都經過磨練。

**「量子諧波」**
哲學：離散實體展現波狀干涉圖案。
演算表達：粒子在網格上初始化，各攜帶一個透過正弦波演化的相位值。粒子相近時，其相位相互干涉——建設性干涉創造亮節點，破壞性干涉創造空洞。簡單諧波運動生成複雜的湧現曼陀羅。這是煞費苦心的頻率校準之果，每個比率都精心選擇以產生共鳴之美。

**「遞歸耳語」**
哲學：尺度間的自相似性，有限空間中的無限深度。
演算表達：遞歸細分的分支結構。每個分支略帶隨機性，但受黃金比例約束。L 系統或遞歸細分生成既有數學感又有機的樹狀形態。細微的噪聲擾動打破完美對稱。線條粗細隨每個遞歸層級遞減。每個分支角度都是深度數學探索的產物。

**「場動力學」**
哲學：通過其對物質的影響使不可見的力可見。
演算表達：由數學函式或噪聲構成的向量場。粒子從邊緣誕生，沿場線流動，在達到平衡或邊界時消亡。多個場可吸引、排斥或旋轉粒子。視覺化只呈現軌跡——不可見力量的幽靈般痕跡。一場透過力的平衡精心編排的運算舞蹈。

**「隨機結晶」**
哲學：隨機過程結晶成有序結構。
演算表達：隨機化的圓堆積或 Voronoi 鑲嵌。從隨機點開始，讓它們透過鬆弛演算法演化。細胞相互推擠至平衡。基於細胞大小、鄰居數量或與中心的距離著色。湧現的有機鑲嵌感覺既隨機又必然。每個種子產生獨特的晶體之美——大師級生成演算法的標誌。

*這些是精簡版範例。實際的演算哲學應為 4-6 段實質性內容。*

### 基本原則
- **演算哲學**：建立一個透過程式碼表達的運算世界觀
- **過程勝於產品**：始終強調美從演算法的執行中湧現——每次運行都是獨特的
- **參數化表達**：想法透過數學關係、力、行為傳達——而非靜態構圖
- **藝術自由**：下一個 Claude 以演算方式詮釋哲學——提供創意實作空間
- **純粹生成藝術**：這是關於創作**活的演算法**，而非帶有隨機性的靜態圖像
- **專家工藝**：反覆強調最終演算法必須感覺精心打造、經過無數次迭代精煉，出自運算美學領域絕對頂尖的大師之手

**演算哲學應為 4-6 段。** 以詩意的運算哲學填充，將預期的願景融為一體。避免重複相同的觀點。將此演算哲學作為 .md 檔案輸出。

## 推導概念種子

**關鍵步驟**：在實作演算法前，從原始請求中識別微妙的概念脈絡。

**核心原則**：
概念是**嵌入演算法本身中的微妙、小眾參照**——不一定是字面意思，始終是精緻的。熟悉此主題的人應能直覺感受到它，而其他人則只是體驗到一件卓越的生成作品。演算哲學提供運算語言。推導出的概念提供靈魂——安靜的概念 DNA 無形地編織進參數、行為和湧現模式中。

這**非常重要**：參照必須如此精煉，使其深化作品的內涵而不宣示自身。像爵士樂手透過演算和聲引用另一首歌——只有懂的人才會捕捉到，但每個人都欣賞生成之美。

## P5.JS 實作

建立好哲學**和**概念框架後，透過程式碼表達它。在繼續前暫停整理思緒。只使用已建立的演算哲學和以下指示。

### ⚠️ 步驟零：先讀取範本 ⚠️

**重要：撰寫任何 HTML 前：**

1. 使用讀取工具**讀取** `templates/viewer.html`
2. **研究**確切的結構、樣式和 Anthropic 品牌
3. **以該檔案作為起點**——不只是靈感
4. **完全保留所有固定區段**（標題、側邊欄結構、Anthropic 顏色/字型、種子控制、操作按鈕）
5. **只替換變數區段**（在檔案注釋中標記）（演算法、參數、參數的 UI 控制）

**避免：**
- ❌ 從頭建立 HTML
- ❌ 發明自訂樣式或配色方案
- ❌ 使用系統字型或深色主題
- ❌ 更改側邊欄結構

**遵循以下做法：**
- ✅ 複製範本的確切 HTML 結構
- ✅ 保留 Anthropic 品牌（Poppins/Lora 字型、淺色配色、漸層背景）
- ✅ 維持側邊欄佈局（種子 → 參數 → 顏色？ → 操作）
- ✅ 只替換 p5.js 演算法和參數控制

範本是基礎。在其上建構，不要重建。

要創作有畫廊品質、有生命力的運算藝術，以演算哲學為基礎。

### 技術需求

**種子隨機性（Art Blocks 模式）**：
```javascript
// 始終使用種子以確保可重現性
let seed = 12345; // 或來自使用者輸入的哈希
randomSeed(seed);
noiseSeed(seed);
```

**參數結構——遵循哲學**：

建立從演算哲學自然湧現的參數，考慮：「這個系統的哪些特質可以調整？」

```javascript
let params = {
  seed: 12345,  // 始終包含種子以確保可重現性
  // 顏色
  // 添加控制你的演算法的參數：
  // - 數量（多少？）
  // - 尺度（多大？多快？）
  // - 概率（多可能？）
  // - 比率（什麼比例？）
  // - 角度（什麼方向？）
  // - 閾值（何時行為改變？）
};
```

**設計有效參數時，專注於系統需要可調整的屬性，而非從「模式類型」的角度思考。**

**核心演算法——表達哲學**：

**重要**：演算哲學應決定要建構什麼。

透過程式碼表達哲學時，避免思考「我應該用哪種模式？」，而是思考「如何透過程式碼表達這套哲學？」

若哲學是關於**有機湧現**，考慮使用：
- 隨時間積累或生長的元素
- 受自然規律約束的隨機過程
- 回饋迴路和互動

若哲學是關於**數學之美**，考慮使用：
- 幾何關係和比率
- 三角函式和諧波
- 精確計算創造意外的模式

若哲學是關於**受控混沌**，考慮使用：
- 嚴格邊界內的隨機變異
- 分岔和相變
- 從無序中湧現的秩序

**演算法從哲學流出，而非從選項選單。**

為引導實作，讓概念本質影響創意和原創的選擇。建構能表達這個特定請求願景的東西。

**畫布設定**：標準 p5.js 結構：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化你的系統
}

function draw() {
  // 你的生成演算法
  // 可以是靜態（noLoop）或動態的
}
```

### 工藝需求

**重要**：要達到精通，創作感覺如同大師生成藝術家經過無數次迭代打磨而來的演算法。仔細調整每個參數。確保每個模式都有目的地湧現。這**不是**隨機噪聲——這是透過深厚專業知識精煉的**受控混沌**。

- **平衡**：複雜但無視覺噪聲，有序但不僵硬
- **色彩和諧**：經過思考的調色板，而非隨機的 RGB 值
- **構圖**：即使在隨機性中，也維持視覺層次和流動
- **性能**：流暢執行，若有動畫則針對即時優化
- **可重現性**：相同種子**始終**產生相同輸出

### 輸出格式

輸出：
1. **演算哲學** — 作為 Markdown 或文字，解釋生成美學
2. **單一 HTML Artifact** — 從 `templates/viewer.html` 建構的自包含互動式生成藝術（見步驟零和下一節）

HTML artifact 包含一切：p5.js（來自 CDN）、演算法、參數控制和 UI——全部在一個可在 claude.ai artifacts 或任何瀏覽器中立即運行的檔案中。從範本檔案開始，不要從頭建構。

## 互動式 ARTIFACT 建立

**提醒：`templates/viewer.html` 應已在步驟零中讀取。以該檔案作為起點。**

要允許探索生成藝術，建立一個單一的自包含 HTML artifact。確保此 artifact 可在 claude.ai 或任何瀏覽器中立即運行——不需要任何設定。內嵌所有內容。

### 重要：固定 vs. 變數內容

`templates/viewer.html` 檔案是基礎。它包含所需的確切結構和樣式。

**固定（始終完全按照顯示包含）：**
- 佈局結構（標題、側邊欄、主畫布區域）
- Anthropic 品牌（UI 顏色、字型、漸層）
- 側邊欄中的種子區段：
  - 種子顯示
  - 上一個/下一個按鈕
  - 隨機按鈕
  - 跳至種子輸入框＋確定按鈕
- 側邊欄中的操作區段：
  - 重新生成按鈕
  - 重置按鈕

**變數（為每件藝術作品自訂）：**
- 整個 p5.js 演算法（setup/draw/類別）
- 參數物件（定義藝術需要什麼）
- 側邊欄中的參數區段：
  - 參數控制的數量
  - 參數名稱
  - 滑桿的最小值/最大值/步進值
  - 控制類型（滑桿、輸入框等）
- 顏色區段（可選）：
  - 部分藝術需要顏色選擇器
  - 部分藝術可能使用固定顏色
  - 部分藝術可能是單色（不需要顏色控制）
  - 根據藝術的需求決定

**每件藝術作品都應有獨特的參數和演算法！** 固定部分提供一致的用戶體驗——其他一切表達獨特的願景。

### 必要功能

**1. 參數控制**
- 數值參數的滑桿（粒子數量、噪聲尺度、速度等）
- 調色板顏色的顏色選擇器
- 參數變更時即時更新
- 重置按鈕以恢復預設值

**2. 種子導覽**
- 顯示當前種子號碼
- 「上一個」和「下一個」按鈕循環種子
- 「隨機」按鈕取得隨機種子
- 輸入框跳至特定種子
- 被請求時生成 100 個變體（種子 1-100）

**3. 單一 Artifact 結構**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js 來自 CDN——始終可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有樣式內嵌——乾淨、極簡 */
    /* 畫布在上，控制在下 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有參數控制 -->
  </div>
  <script>
    // 所有 p5.js 程式碼內嵌在這裡
    // 參數物件、類別、函式
    // setup() 和 draw()
    // UI 處理器
    // 一切自包含
  </script>
</body>
</html>
```
